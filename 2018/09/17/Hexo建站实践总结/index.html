2. 实践过程知识点

2.1. 博客类产品的设计

HEXO是属于开源博客类的产品。他的设计成功之处有：

  1. 代码开源
    GITHUB上开发，吸引众多伙伴参与建设；
  2. 网站主题很多
    主题多，可以定制各种漂亮的BLOG。
  3. 安装更新方便
    和GIT结合使用，nodejs的成熟的npm软件包管理，可以非常方便的安装和更新
  4. 使用简便
    安装后，BLOG使用流行的MarkDown语言编写，写起来比较惬意。

3. 安装和部署过程

总体安装过程比较简单，部署也比较容易：

  1. 安装包：node.js 、git
    安装node.js过程中，自动安装npm，同时也可以自动安装编译nodejs 所需要的python VS2017工具等；
  2. 编辑器：VSCODE
    用作makerdown的编辑器
  3. HEXO静态博客框架安装和初始化
    在git bash 中，使用$npm install -g hexo-cli 命令安装hexo框架。空白博客，使用hexo init初始化，过程比较慢，需要下载依赖，可能要多试几次。
  4. hexo的next主题安装
    安装next主题库git clone git@github.com:theme-next/hexo-theme-next.git next。
  5. github page网站仓库建立
    博客写好后，可以直接上传到github page上。我的github page
  6. 使用git bash编译和部署博客
    初始化Github page，然后使用SSH生成本地公钥和私钥，同步到github 上，就可以自由的更新博客了，具体命令见4.3节 hexo常用命令。

5. 相关知识点

5.1 yaml文件结构和配置

YAML（Yet Another Markup Language）（发音 /ˈjæməl/ ）一种基于Unicode容易阅读，容易和脚本语言交互的，用来表达资料序列的编程语言。YAML文件简介 这个文件基本介绍的比较详细。
语法：

  • 使用空格 Space 缩进表示分层，不同层次之间的缩进可以使用不同的空格数目，但是同层元素一定左对齐，即前面空格数目相同（不能使用 Tab，各个系统 Tab对应的 Space 数目可能不同，导致层次混乱）;
  • ‘#’表示注释，只能单行注释，从#开始处到行尾;
  • 破折号后面跟一个空格（a dash and space）表示列表;
  • 用冒号和空格表示键值对 key: value；
  • 简单数据（scalars，标量数据）可以不使用引号括起来，包括字符串数据。用单引号或者双引号括起来的被当作字符串数据，在单引号或双引号中使用C风格的转义字符；

简单说：就是一种相对简单的标记语言，使用空格和缩进表示分层，同一元素一定左对齐。

5.2. ssh通信基础 (git免密上传）

SSH为Secure Shell的缩写，由IETF的网络工作小组（Network Working Group）所制定；SSH为创建在应用层和传输层基础上的安全协议。基本原理概念介绍，这篇文章写的非常好： 网络数据传输安全及SSH与HTTPS工作原理

先看这篇的总结也不错：SSH原理和使用
需要理解的是， SSH就是一种网络协议，主要用于计算机之间的加密登录与数据传输。SSH是充分利用了公钥加密/非对称机密 、对称加密 和 单向加密来实现数据安全。
