    Mysql workbench 使用实践

Select查询后，有时候result grid 不出现，重启Workbench解决；

  • 执行选中的SQL(如无选中则执行所有) Ctrl + Shift + Enter
  • 执行当前这句SQL (注意MySQL Workbench的编辑器会要求每一句SQL必须以;结尾, 否则会高亮提示错误) Ctrl + Enter
  • 注释掉选中SQL Ctrl + /
  • 格式化SQL Ctrl + B
  • 新开SQL编辑器 Ctrl + T
  • 查看执行解释 (explain current statement) Ctrl + Alt + X

.Net GC垃圾回收原理

Posted on 2018-09-22 | Edited on 2018-10-11 | In .net | Comments: | Views:

.net 的垃圾回收机制GC

对象的生命

一个对象的生命周期简单概括就是：创建>使用>释放，在.NET中一个对象的生命周期：

  1. new创建对象并分配内存；
  2. 对象初始化；
  3. 对象操作、使用；
  4. 资源清理（Managed Resources）；
  5. GC垃圾回收（Unmanaged Resources）；

用C++进行编程，内存的处理绝对是让每个程序设计者最头疼的一块了。但是对于.net平台下使用c#语言开发系统，内存管理可以说已经不算是问题了。在.net平台下CLR负责管理内存，CLR中的垃圾收集器GC：Garbage Collection，负责执行内存的清理工作，但是GC也只是负责清理托管堆上的垃圾对象，而对于非托管的资源对象，GC则不起作用，必须要程序开发者手动清理。对于Stream，数据库的连接，GDI+的相关对象，还有Com对象等等，这些资源并不是受到.Net管理而统称为非托管资源，非托管资源需要手动显式进行释放。
