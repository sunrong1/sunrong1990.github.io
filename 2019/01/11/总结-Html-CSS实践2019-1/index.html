Html CSS的关系

在Web开发中，html 语言给Web网页的内容提供了组织结构和语义（每个部分的作用），CSS控制了内容的布置方式和外观，描述怎么显示Html文档。
In web development, HTML gives structure and semantics to a page’s content, and CSS controls the layout and appearance of it.
This section covers some of the basic tools developers use to create their own visual designs.
使用CSS的优势：

  1. 比Html提供更强大的工具，对页面内容进行展示；
  2. 内容和展示分离，使得一个CSS文件可以用于多个网页的展示，开发维护方便，更新方便。
  3. 分离的另一个好处，给用户提供个性化显示方案，用户可以自由选择不同样式。
  4. 可提高页面加载速度

CSS规则

  1. A CSS rule is broken into two parts: the selector and the property。
  1. 选择CSS选择器时，需要考虑：
  • How many of these HTML elements are on my page? Do I want this style to apply to every one of these elements?
  • What are this HTML element’s children, and do I want this style to apply to them as well?
  • Is this element a block element or an inline element, and does this style make sense in that context?
  1. 书写CSS代码时，注意属性是否继承，同时建议先熟悉Html文档的结构，然后再决定CSS的顺序：
    Not every property is inherited, but many are. The CSS specification tell you, for each property, whether it is inheritable. It’s a good idea to keep in mind the structure of your HTML document when choosing your selectors so you can use inheritance to your advantage by applying styles to the top most element and save yourself extra CSS code.

CSS选择器

合理使用使用CSS选择器，可以使我们HTML的代码更清洁，减少一些不必要的Clss和id的使用。456Berea—更充分使用选择器

参考阮一峰博客

  1. 基本选择器
序号 选择器 含义
1. * 通用元素选择器，匹配任何元素
2. E 标签选择器，匹配所有使用E标签的元素
3. .info class选择器，匹配所有class属性中包含info的元素
4. #footer id选择器，匹配所有id属性等于footer的元素
  1. 多元素的组合选择器
序号 选择器 含义
5. E,F 多元素选择器，同时匹配所有E元素或F元素，E和F之间用逗号分隔
6. E F 后代元素选择器，匹配所有属于E元素后代的F元素，E和F之间用空格分隔
7. E > F 子元素选择器，匹配所有E元素的子元素F
8. E + F 毗邻元素选择器，匹配所有紧随E元素之后的同级元素F
  1. CSS 2.1 属性选择器
序号 选择器 含义
9. E[att] 匹配所有具有att属性的E元素，不考虑它的值。（注意：E在此处可以省略，比如”[cheacked]”。以下同。）
10. E[att=val] 匹配所有att属性等于”val”的E元素
11. E[att~=val] 匹配所有att属性具有多个空格分隔的值、其中一个值等于”val”的E元素
12. E[att\ =val] 匹配所有att属性具有多个连字号分隔（hyphen-separated）的值、其中一个值以”val”开头的E元素，主要用于lang属性，比如”en”、”en-us”、”en-gb”等等
  1. CSS 2.1中的伪元素
序号 选择器 含义
20. E:first-line 匹配E元素的第一行
21. E:first-letter 匹配E元素的第一个字母
22. E:before 在E元素之前插入生成的内容
23. E:after 在E元素之后插入生成的内容
